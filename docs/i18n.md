Adding internationalization (i18n) to a React application like ConClár, which currently relies heavily on a `config.json` file for its UI strings, requires a structured approach. The best practice for React applications is to use the `react-i18next` library, which is the React wrapper for the industry-standard `i18next` framework.

#### Best Practice: `react-i18next`

`react-i18next` is recommended because:
- **Ease of use:** It provides hooks like `useTranslation` and components for handling pluralization, formatting, and nesting.
- **Scalability:** It makes adding new languages (like German after English) as simple as adding a new JSON translation file.
- **Rich ecosystem:** It supports language detection (browser-based), loading translations from backend servers, and caching.

---

### How to Start (Step-by-Step)

#### 1. Install Dependencies
You will need the following packages:
```bash
npm install i18next react-i18next i18next-browser-languagedetector
```

#### 2. Create Translation Files
Create a new directory structure for your translations, for example: `src/locales/`.

**`src/locales/en.json`**
```json
{
  "navigation": {
    "program": "Programme",
    "people": "People",
    "myschedule": "My Schedule"
  },
  "settings": {
    "title": "Settings",
    "darkMode": "Dark Mode"
  }
}
```

**`src/locales/de.json`**
```json
{
  "navigation": {
    "program": "Programm",
    "people": "Teilnehmende",
    "myschedule": "Mein Plan"
  },
  "settings": {
    "title": "Einstellungen",
    "darkMode": "Dunkelmodus"
  }
}
```

#### 3. Initialize i18next
Create an `i18n.js` file in your `src` directory to configure the library.

**`src/i18n.js`**
```javascript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import en from './locales/en.json';
import de from './locales/de.json';

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: en },
      de: { translation: de }
    },
    fallbackLng: 'en',
    interpolation: {
      escapeValue: false // React already escapes values
    }
  });

export default i18n;
```

Import this file in your `src/main.jsx` (or `src/index.js`) so it initializes when the app starts.

#### 4. Replace Static Strings in Components
In your components (like `src/components/Settings.jsx` or `src/components/People.jsx`), replace the `configData` references with the `useTranslation` hook.

**Example: `src/components/People.jsx`**
```jsx
import { useTranslation } from 'react-i18next';

const People = () => {
  const { t } = useTranslation();
  // ...
  return (
    <div className="people-search">
      <label htmlFor="people-search">{t('people.search_label')}</label>
      {/* ... */}
    </div>
  );
};
```

#### 5. Integrate with State Management (`easy-peasy`)
Since your app uses `easy-peasy`, you should store the current language in your store model (`src/store/model.js`). This allows you to persist the user's language choice in `localStorage` and trigger re-renders across the app when the language changes.

- **Add a `language` state** to your model.
- **Add a `setLanguage` action** that calls `i18n.changeLanguage(lang)`.

---

### Key Recommendations for ConClár
1. **Gradual Migration:** Since many labels are currently in `config.json`, start by moving those labels into your `en.json` and `de.json` files. This will clean up your configuration and make the UI truly dynamic.
2. **Handle Dynamic Content:** The `program.js` and `people.js` files (like `bio` or `description`) are currently monolingual. To support internationalization for this data, you may need to update your data schema to include multi-language fields (e.g., `bio_en`, `bio_de`) and update `ProgramData.js` to select the correct one based on the active language.
3. **Language Switcher:** Add a dropdown or toggle in the `Settings.jsx` component to allow users to manually switch between English and German.
